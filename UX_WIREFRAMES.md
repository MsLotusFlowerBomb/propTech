## Design System

### Color Palette

#### Primary Colors
- **Brand Primary**: `#2196F3` (Blue) - Trust, professionalism
- **Brand Secondary**: `#FF9800` (Orange) - AI features, highlights
- **Brand Accent**: `#4CAF50` (Green) - Success, positive actions

#### Semantic Colors
- **Success**: `#4CAF50` (Green)
- **Warning**: `#FFC107` (Amber)
- **Error**: `#F44336` (Red)
- **Info**: `#2196F3` (Blue)

#### Neutral Colors
- **Background**: `#FAFAFA` (Light Gray)
- **Surface**: `#FFFFFF` (White)
- **Border**: `#E0E0E0` (Gray 300)
- **Text Primary**: `#212121` (Gray 900)
- **Text Secondary**: `#757575` (Gray 600)

### Typography

#### Font Family
- **Primary**: Segoe UI (Windows), SF Pro (macOS), Roboto (Web)
- **Monospace**: Consolas, Monaco, Courier New

#### Font Scales
- **H1**: 32px / 2rem - Page titles
- **H2**: 24px / 1.5rem - Section headers
- **H3**: 20px / 1.25rem - Subsection headers
- **H4**: 18px / 1.125rem - Card titles
- **Body**: 16px / 1rem - Main content
- **Small**: 14px / 0.875rem - Secondary text
- **Tiny**: 12px / 0.75rem - Captions, labels

### Spacing System
- **XS**: 4px
- **SM**: 8px
- **MD**: 16px
- **LG**: 24px
- **XL**: 32px
- **XXL**: 48px

### Component Library

#### Buttons
- **Primary**: Filled button, brand primary color
- **Secondary**: Outlined button
- **Tertiary**: Text button
- **Icon**: Icon-only button
- **FAB**: Floating action button

#### Cards
- **Standard**: White background, subtle shadow
- **Elevated**: Larger shadow for hierarchy
- **Outlined**: Border instead of shadow
- **Interactive**: Hover effects, clickable

#### Forms
- **Text Input**: Single line text field
- **Text Area**: Multi-line text field
- **Select**: Dropdown selector
- **Checkbox**: Multiple selection
- **Radio**: Single selection
- **Toggle**: Binary switch
- **Date Picker**: Calendar selector
- **File Upload**: Drag-and-drop zone

## Accessibility Features

### 1. Keyboard Navigation
- Tab order follows logical flow
- All interactive elements accessible via keyboard
- Escape closes modals/dialogs
- Enter submits forms

### 2. Screen Reader Support
- ARIA labels on all interactive elements
- Alt text on images
- Semantic HTML structure
- Announcement of dynamic content changes

### 3. Visual Accessibility
- Minimum contrast ratio 4.5:1 for text
- Text resizable up to 200%
- No information by color alone
- Focus indicators on all interactive elements

### 4. Touch Targets
- Minimum 44x44px for touch targets
- Adequate spacing between interactive elements
- Swipe gestures for common actions (mobile)

---

## Responsive Breakpoints

- **Mobile**: < 768px (single column, bottom nav)
- **Tablet**: 768px - 1024px (2 columns, mixed nav)
- **Desktop**: > 1024px (sidebar nav, multi-column)

---

## Animation & Motion

### Principles
- **Meaningful**: Animations guide attention
- **Fast**: < 300ms for most transitions
- **Subtle**: Don't distract from content

### Examples
- **Page transitions**: Fade in (150ms)
- **Modal appearance**: Scale up + fade (200ms)
- **List items**: Stagger animation (50ms delay each)
- **Loading**: Spinner or skeleton screens
- **Success**: Check mark animation (300ms)

## Conclusion

This UX wireframe documentation provides a comprehensive view of all key screens and interactions in the PropMate platform. The design prioritizes:

1. **Clarity**: Information hierarchy and scanability
2. **Efficiency**: Common tasks require minimal clicks
3. **Intelligence**: AI insights prominently displayed
4. **Trust**: Transparency in AI predictions and processes
5. **Accessibility**: Usable by all users regardless of ability

The dual-interface strategy (MAUI desktop + Blazor web) ensures optimal experience for each user type while maintaining consistent branding and functionality.
